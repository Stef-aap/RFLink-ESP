Raw detection

RawSignal.Repeats is of great importance, because if it's true, detection is blocked.

RawSignal.Repeats is used both as an integer(byte) and a boolean ???

RawSignal.Repeats is removed from the detection, might still be needed by the RF generation as a repeat-count.

 

 

LoopsPerMilli = 2500;

For this value the following switches work well:

  • Notenhouten A/B/C/D button  (EVT1527)
  • Ventilator 4 button ( EVT1527 ?)
  • KALU keuken (KAKUnew)

 

The lowest / highest values at which all these switches works well = 1500 / 4300. That's why we choose 2500 which is about the mid value.

The 6 button is over whole range unreliable, especially the off-button manifest itself as:

 

 

Note that not only On/Off is wrong, but also the ID changes

So the PT2262 needs improvement !!!

 

The above measurements are made with the following settings:

  • Start-protocol is disabled
  • Start detection in FetchSignal is disabled
  • RawSignal.Repeats is removed

 

Changing the above settings back to the orginal, doesn't improve the PT2262 detection !!!
